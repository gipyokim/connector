2. 공통

다음 옵션들은 모든 덤프 대상에서 사용할 수 있다.

옵션설명

type (필수)

작업 종류

  • fsg : 파일 시스템 덤프

  • rfsg : 원격 파일 시스템 덤프(2.1.11부터 지원)

  • dbg : 데이터베이스 덤프

  • reproc : 재처리

  • mongo : MongoDB 덤프

  • sps : SharePoint 덤프

  • notes : Notes 덤프

  • xml : XML 파일 덤프

format

덤프 파일 종류 (기본값 : fgf)

  • fgf | txt

charset

덤프 파일 문자 셋 (기본값 : UTF-8)

  • UTF-8 | EUC-KR

max_file_size

덤프 파일 교체 크기 (기본값 :10MB)

KB, MB, GB 단위로 설정할 수 있다. 단위가 없을 때는 바이트로 간주한다.

참고

대상 파일에 로그파일이나 압축파일 등이 포함된 경우, 필터된 TEXT 용량이 설정된 max_file_size보다 클 수 있다.

target_dir (필수)

덤프 파일 경로

init

작업 시작 전 덤프 파일 경로의 기존 파일 처리 방법 (기본값 :clear)

  • clear : 파일 삭제

  • backup : 파일 백업

    참고

    백업된 파일들은 target_dir 하위 backup 디렉터리에 원본 "디렉터리명"으로 저장된다.

  • keep : 파일 유지

queue_size

대기큐 크기 (기본값 :100)

thread_count

작업 스레드 개수 (기본값 :4)

before

덤프 전 실행 작업

사용자 정의 그루비 메소드를 작성하여 덤프 전 진행해야 할 특별한 작업을 수행한다.

  • 그루비 메소드 String beforeProcess(String mode))사용

    before = SampleGroovy.beforeProcess("bulk")
    before = SampleGroovy.beforeProcess(@mode@)

    before, after에서는 @로 인자를 묶는 파라미터의 사용이 가능하다.

    @type@, @name@, @mode@ 가 주로 사용되며, system.log에 테이블로 출력되는 모든 key를 파라미터로 사용할 수 있다.

  • 그루비 메소드의 자세한 사용 방법은 "beforeProcess"를 참고한다.

after

덤프 후 실행 작업

사용자 정의 그루비 메소드를 작성하여 덤프 후 진행해야할 특별한 작업을 수행한다.

  • 그루비 메소드 String afterProcess(String mode)) 사용

    after = SampleGroovy.afterProcess("bulk")
    after = SampleGroovy.afterProcess(@mode@)

  • 사용 방법은 before 옵션과 동일하다.

allow_file_types

필터링할 확장자 정의

allow_file_types = txt,doc,hwp

deny_file_types

필터링 시 제외할 확장자 정의

deny_file_types = bmp,jpg,gif,
ico,img,png,psd,vsd,tif,tiff,
mpg,mpeg,mp3,mp4,ra,ram,ogg,
avi,asf,swf,wmv,mov,wmf,wma,
emf,wav,exe,exp,msi,bat

참고

allow_file_types와 deny_file_types는 서로 배타적으로 동작된다. deny_file_types가 설정되어 있으면 allow_file_types는 동작하지 않으며 그 반대도 마찬가지이다.

확장자가 없는 파일을 지정하려면 빈 확장자를 설정한다.

deny_file_types = bmp,

위와같이 지정하면 bmp와 확장자가 없는 파일은 제외된다. 확장자가 없는 파일만을 지정하는 방법은 지원하지 않는다.

text_filter

텍스트 필터 (기본값 :charset=UTF-8)

text_filter TextFilter = {
    address = 127.0.0.1:7574
    charset = UTF-8
}

  • address : 필터링 서버 주소 (host:port)

  • charset : 필터링 서버 요청 시 사용할 문자 셋으로, 생략하면 UTF-8로 동작한다.

    참고

    Unix, Linux 서버에 필터 서버가 설치되어 있고 시스템 문자 셋이 EUC-KR이며, 또한 필터링할 한글 파일명이 존재할 경우에 EUC-KR로 설정한다. 그 외 charset 설정은 생략해도 된다.
