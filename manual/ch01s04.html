4. 용어 설명

Bulk (벌크)

원본 데이터를 일괄 수입(import) 하여 덤프 파일로 변환한다.

Incremental (증분)

원본 데이터의 추가, 변경, 삭제된 내용만을 수입하여 덤프 파일로 변환한다.

High Value Mark (HVM)

증분 덤프 방식의 하나로 특정 날짜 칼럼의 최댓값을 기록하고 있다가, 덤프 시 기록된 날짜 이후의 데이터만을 가져오는 방식이다.

Log Table

원본 테이블의 데이터 변경 이벤트에 대해 트리거를 걸어 변경 이력을 LOG 테이블에 저장한 후 증분 덤프 시 이 변경 이력을 이용해 덤프하는 방식이다.

덤프 파일 (Dump File)

커넥터를 이용하여 원본 데이터를 수집, 가공하여 저장한 파일을 의미하며 검색엔진이 읽을 수 있도록 변환된 파일이다.
