1. 문법

작업 설정 파일(*.rc)은 아래와 같은 문법으로 작성한다.

# Comment
 
# Include directive
!include file_path
 
# Definition
key_1 = value_1
 
# Multi-line definition
key_2 = line_1 \
line_2 \
line_3
 
# Inline definition
key_3 = {
    sub_key_31 = value_31
    sub_key_32 = value_32
}
 
# Declaration
key_4 name = {
    sub_key_41 = value_41
    sub_key_42 = value_42
}
 
# Reference
key_4 = name

공백 라인들은 무시되며, 샾(#) 문자로 시작되는 라인은 주석으로 취급된다. 느낌표(!)로 시작하는 라인은 명령어로 현재 include만 지원된다.

나머지 라인들은 각종 옵션에 대한 정의로서 단일 값 정의 및 속성 정의로 나누어진다. 역슬래시(\) 문자를 라인 끝에 붙일 경우 여러 라인의 값을 정의할 수 있다. 속성 정의는 key = {}로 정의하며 라인 끝에 '{' 와 '}'가 위치해야 한다. 아래와 같이 속성을 먼저 선언하고 뒤에서 참조할 수도 있다.

  • 속성 선언

    key_4 name = {
    sub_key_41 = value_41
    sub_key_42 = value_42
}

  • 참조

    key_4 = name

    참고

    위처럼 name으로 속성 선언을 했다면 반드시 뒤에서 참조해야 정상적으로 동작된다.

key_4는

key_4 = {
    sub_key_41 = value_41
    sub_key_42 = value_42
}

와 같은 효과를 본다. 같은 속성을 여러 옵션으로 정의할 경우 유용하다.
