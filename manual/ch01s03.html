3. 주요 특징

추가 기능

  • 덤프 작업 시 예상 완료 시각을 조회할 수 있다.

  • 파일 시스템 덤프 작업 시 파일 목록 스캐닝 작업과 텍스트 필터링 작업을 분리하여 수행할 수 있다.

  • Microsoft SharePoint, IBM Notes 덤프 기능이 추가되었다. (2.0.1부터 지원)

  • Microsoft SharePoint, IBM Notes 덤프 작업에 대한 쿼리 테스트 기능이 추가되었다. (2.0.1부터 지원)

  • 작업 테스트 시 그루비 스크립트에 대한 테스트도 포함할 수 있다. (2.0.1부터 지원)

  • 증분 덤프 후 추가, 변경, 삭제 레코드 개수도 기록한다. (2.0.1부터 지원)

  • XML 파일 덤프 기능이 추가되었으며 XML 1건당 레코드 1건을 덤프한다. (Bulk 덤프만 지원함) (2.0.2부터 지원)

  • 파일 시스템 덤프 설정 시 field_mapping을 지원한다. (2.0.3부터 지원)

  • 데이터베이스 High Value Mark 증분 덤프 설정 시 소프트/하드 파싱을 선택할 수 있다. (2.0.3부터 지원)

  • MongoDB 덤프 기능이 추가되었다. (2.0.4부터 지원)

  • MongoDB 덤프 작업에 대한 쿼리 테스트가 추가되었다. (2.0.4부터 지원)

  • 로그 테이블 기반 증분 덤프 설정 시 커스텀마이징된 로그테이블명과 칼럼명을 지원한다. (2.0.4부터 지원)

  • Notes 서비스에 대한 병렬 처리를 지원한다. (2.0.4부터 지원)

  • XML 증분 덤프 기능이 추가되었다. (2.1.1부터 지원)

  • Notes(local) 덤프 시 DIIOP 통신을 통해 추가적인 설치없이 덤프를 받을 수 있는 기능이 추가되었다. (2.1.3부터 지원)

  • DB 정보 조회 기능이 추가되었다. (2.1.4부터 지원)

  • 덤프 시 특정 레코드를 제외하는 기능이 추가되었다. (2.1.6부터 지원)

  • query 설정 시 multiline String 기능이 추가되었다. (2.1.6부터 지원)

  • 원격 파일 필터링 기능을 지원한다. (2.1.11부터 지원)

  • 로그 테이블 기반 증분 덤프 설정 시 증분 덤프와 로그 테이블 레코드 삭제 기능의 분리를 지원한다. (2.1.11부터 지원)

변경 및 개선 기능

  • 파일 시스템 덤프 작업 시 소스 디렉터리 패스명의 길이 제약이 사라졌다.

  • 파일 시스템 덤프 작업 시 실패한 파일의 경로를 별도의 로그 파일로 기록한다.

  • 데이터베이스 덤프 작업 설정 시 필드 매핑을 생략할 수 있다.

  • 데이터베이스 덤프 병렬 처리 시 서브 작업에 대해 파라미터 설정을 지원한다.

  • 텍스트 필터링 시 제외할 파일 확장자를 설정할 수 있다.

  • 재처리 작업 설정 시 소스 필드 매핑이 제거되었다.

  • 더는 내장 데이터베이스를 필요로 하지 않는다.

  • Check progress 설정은 monitoring interval 설정으로 변경되었다.

    Check progress 설정은 몇 개의 레코드가 처리될 때마다 로그에 기록할지를 결정하였다. 따라서 일정 개수의 레코드가 처리되기 전까지는 로그 파일에 변경이 없으므로 사용자는 덤프 작업이 정상적으로 진행 중인지 확인하기 어려웠다. Monitoring interval 설정은 레코드 개수와 관계없이 일정 시간 간격에 따라 진행 상태를 콘솔 창이나 로그 파일에 기록하게 한다. 사용자는 현재까지 처리된 레코드 개수를 확인하여 덤프 작업이 정상적으로 진행 중인지 확인할 수 있다. Monitoring interval 및 로그 파일로의 출력 여부 설정은 "덤프"에서 -mi, -q 옵션을 참고하기 바란다.

  • 시스템 로그 파일 생성을 크기 기반에서 날짜별 생성으로 변경하였다. (2.0.3부터 지원)

  • 라이선스 검사가 제거되었다. (2.1.4부터 지원)

  • 덤프 옵션 중 -q 옵션이 제거되고, -v 옵션이 추가되었다. (2.1.6부터 지원)

1.6 버전 대비 미구현 기능

  • 필드 매핑 시 JDBC Property는 더는 지원하지 않는다.

  • 덤프 파일 포맷은 FGF와 TXT를 지원한다. XML, JSON 형식의 덤프 파일은 지원하지 않는다.
