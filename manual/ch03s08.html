8. Notes 덤프

다음 옵션들은 IBM Notes 덤프 작업 시 사용하는 설정이다.

옵션설명

notes_source (필수)

Notes 데이터 소스

  • Local Access

    notes_source = {
    server = 10.10.20.29
}

  • Remote Access

    notes_source = {
      server = 10.10.20.92
      username = konan
      password = konan415
      iiop     = true
}

include

수집 대상 nsf 파일 또는 디렉터리

여러 개를 지정할 경우 콤마(,)로 구분한다.

include = QA01/NEW/, QA01/QATEAM/QATEAM01/

exclude

수집 제외 nsf 파일 또는 디렉터리

여러 개를 지정할 경우 콤마(,)로 구분한다. 

exclude = QA01/NEW308/

formula

Domino 서버에서 해당 formula로 설정된 대상 수집

view

Domino 서버에서 해당 view로 설정된 대상 수집

fetch_size

한 번에 패치할 레코드 개수 (기본값 : 1024)

field_mapping (필수)

"필드 매핑"을 참고한다.

참고

  • 예외값 : 공백

temp_dir

첨부 파일을 내려받을 임시 디렉터리 경로 (기본값 : target_dir/temp)

limit_count

데이터 덤프 개수 지정

limit_count = 10

skip

기존에는 notes 증분 시, 삭제된 nsf를 보고 delete 코드를 생성했다. 2.0.7 버전 부터는 skip=ture로 설정하면 nsf에 대한 삭제 코드를 생성되지 않는다. 따라서, nsf 파일을 열지 못하는 경우 파일을 삭제하지 않고 스킵한다.

2.1.11 버전 부터 기본값이 false에서 true로 변경됨.

skip = true
