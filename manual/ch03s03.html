3. 파일시스템

다음 옵션들은 파일시스템 덤프에 사용하는 설정이다.

옵션설명

source_dir (필수)

덤프 대상 디렉터리 경로

여러 디렉터리를 덤프할 경우 역슬래시(\) 문자로 개행하여 나열한다.

source_dir = /data/2012 \
/data/2013 \
/data/2014

deny_dirs

덤프 대상에서 제외할 디렉터리명 패턴으로 아래 네 가지 문법을 콤마(,)로 묶어 설정한다.

  • 패턴* : 패턴으로 시작되는 디렉터리명의 경우 덤프 대상에서 제외

  • *패턴 : 패턴으로 끝나는 디렉터리명의 경우 덤프 대상에서 제외

  • *패턴* : 패턴을 포함하는 디렉터리명의 경우 덤프 대상에서 제외

  • 패턴 : 패턴과 같은 이름의 디렉터리일 경우 덤프 대상에서 제외

deny_dirs = *.swp, .svn*, *temp*, test

field_mapping

"필드 매핑"을 참고하십시오.

참고

  • 예외값 : 공백

  • 설정하지 않을 경우 전체 예약 칼럼과 동일하게 필드명이 매핑된다.

filter_method

필터링 메소드

필터링할 파일의 전체 경로가 파라미터로 전달된다. 소스파일명.메소드명으로 선언한다.

filter_method = 
SampleGroovy.fileFiltering
rtm_url (2.1.11부터 지원)

원격 파일시스템(rfsg)에 한하여 사용 가능하다.

원격지에 설치된 rtm agent의 url을 입력한다.

rtm_url = rtm://localhost:7571
