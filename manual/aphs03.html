3. MySQL - OutOfMemoryError가 발생합니다.

MySQL은 기본적으로 쿼리 결과가 모두 메모리로 로딩됩니다. 대부분의 경우 이러한 방식이 경제적이지만, 테이블에 저장된 레코드가 너무 많을 경우에는 아래와 같이 OutOfMemoryError가 발생할 수도 있습니다.

Retrieving data...
Ran out of memory at row: 0
java.lang.OutOfMemoryError: Java heap space
at ...

이러한 경우에는 Java heap size를 조정하거나 fetch_size 값을 조정하여 문제를 해결할 수 있습니다. 아래와 같이 fetch_size를 -2147483648(Integer.MIN_VALUE)로 설정하면 JDBC 드라이버로 하여금 쿼리 결과를 메모리로 올리지 않고 스트리밍으로 전달하도록 합니다.

fetch_size = -2147483648
